Thing 1: Using " " VS ">" with multiple CSS selectors

A space between two selectors means "apply the following CSS code to ALL <second selector> inside the <first selector>". Specifically, the space defines the second selector as a descendant selector and causes all elements (all children and their subchildren, and their subchildren and ...) in the first selector to be affected by the CSS code. The following example displays this concept.

HTML

<div class="page1">

    <p class="space">
        Outside div
    </p>

    <div class="container">
        <p class="space">
            Inside div
        </p>
    </div>
	
</div>

CSS

.page1 p.space {
    color: red;
    font-size: 1.3em;
    font-weight: 900;
}

Output

Outside div

Inside div

On the contrary, a ">" between two selectors means "apply the following CSS code to all <second selector> inside the <first selector> not enclosed in another tag. Specifically the ">" defines the second selector as a child selector and causes all immediate children (all children, but not their subchildren) in the first selector to be affected by the CSS code. The following example displays this concept.

HTML

<div class="page1">

    <p class="greaterThan">
        Outside div
    </p>

    <div class="container">
        <p class="greaterThan">
            Inside div
        </p>
    </div>
	
</div>

CSS

.page1>p.greaterThan {
    color: red;
    font-size: 1.3em;
    font-weight: 900;
}

Output

Outside div

Inside div

Note: "selector1>selector2" and "selector1 > selector2" mean different things in some browsers because the space sometimes throws browsers off into thinking it's a descendent selector, so always stick to the no-space variant.

Thing 2: The CSS Box Model

For those confused with the difference between padding and margin and what they do, this may help! Imagine a box like this around every single HTML element, with the content of the element (i.e. text, image) inside the content box in this model. Directly outside the content is padding, which is used to position the content inside its border without directly disrupting other things on the page. If you do want to disrupt other things and position your object in respect to its outer container (outside the border), you would use margin. The following example displays this concept.

HTML

<div class="page2">
    Margin
    <div class="main">
        Padding
        <p class="content">
            Content
        </p>
    </div>

    <!-- &uarr; is the up arrow! -->
    &uarr; Border &uarr;
</div>

CSS

.main {
    margin-top: 20px;
    padding: 10px 30px;
    background-color: yellow;
    border: solid darkorange 10px;
}

Output

Margin
Padding

Content

↑ Border ↑

Thing 3: Browser Rendering Debacle

The state of CSS cross-browser compatibility is kind of a mess right now, as browsers like Chrome, Firefox, and IE all tried to implement their own different versions of some new CSS3 properties and functions.

All browsers have their own rendering engine for web pages. Chrome (and Opera) use Google's Blink engine, IE uses Trident, Firefox uses one named Gecko, and Safari uses the good ol' WebKit (Chrome and Opera initially also used WebKit but have since changed to use Blink instead). Thus, for example, the opacity property may be different from the -webkit-opacity or the -moz-opacity or etc.

To make sure that your CSS renders correctly for all browsers, you must use ALL the various rendering engine prefixes along with the original property/function for when it gets officially released.

HTML

<div class="page3">

    <div class="gradient gradient-webkit">
        This background should work for 
        (the most recent versions of): 
        Chrome, 
        Safari, 
        Edge, 
        Firefox,
        and probably Opera.
    </div>

    <div class="gradient gradient-moz">
        This background should only work for:
        Firefox.
    </div>

    <div class="gradient gradient-ms">
        This background should only work for:
        Internet Explorer
        and Edge.
    </div>

</div>

CSS

.gradient {
    width: 500px;
    margin: 10px;
    font-size: 0.8em;
    line-height: 27px;
    padding: 10px;
}

.gradient-webkit {
    background-image: 
    -webkit-linear-gradient(
        top, palegreen, skyblue
    );
}

.gradient-moz {
    background-image: 
    -moz-linear-gradient(
        top, palegreen, skyblue
    );
}

.gradient-ms {
    background-image: 
    -ms-linear-gradient(
        top, palegreen, skyblue
    );
}

Output

This background should work for (the most recent versions of): Chrome, Safari, Edge, Firefox and probably Opera.
This background should only work for: Firefox.
This background should only work for: Internet Explorer and Edge.

Also note that the    ------------- VS -------------    bar in Thing 1 breaks for Internet Explorer and Microsoft Edge!
